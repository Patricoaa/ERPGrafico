### Top 5 problemas críticos
1. **Acoplamiento de Lógica en UI/Shared**: Varios componentes en `/ui` o `/shared` importan páginas o módulos específicos, rompiendo la reusabilidad. (Ej. componentes UI atados a la API de ventas).
2. **"use client" Innecesario**: Muchos componentes y primitivos están forzando "use client" sin usar hooks o estado, lo que degrada el rendimiento del servidor.
3. **Falta de Tipado Estricto**: Uso excesivo de props `any` o tipos parciales en lugar de interfaces completas.
4. **Hardcodeo de UI (Inconsistencia)**: Algunos componentes usan colores o tamaños con clases arbitrarias en Tailwind, ignorando los tokens del diseño base.
5. **Rutas Relativas Largas / Falta de Barrels**: Se están realizando imports de `../../../` sin utilizar alias y faltan `index.ts` de exportación para compilar módulos de forma limpia.

### Estructura recomendada
```
src/components/
├── ui/          ← primitivos shadcn extendidos (intocables) (Ej: Button, Input)
├── shared/      ← compuestos reutilizables entre módulos (Ej: BaseModal, LoadingFallback)
└── modules/     ← componentes específicos de cada módulo
    ├── dashboard/
    ├── inventory/
    ├── sales/
    ├── hr/
    ├── reports/
    └── settings/
```

### Roadmap de refactorización
- **Semana 1: Quick wins** - Limpiar "use client" innecesario, arreglar imports largos y carpetas de barril.
- **Semana 2: Componentes críticos compartidos** - Ajustar los top 5 componentes más usados (botones, tablas, inputs) para usar 100% shadcn.
- **Semana 3: Estandarización por módulo** - Mover componentes desperdigados a su módulo específico dentro de /modules.
- **Semana 4: Documentación y contratos de props** - Completar tipado de todos los componentes quitando los "any" restantes.
